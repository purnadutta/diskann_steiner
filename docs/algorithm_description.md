We augment the Vamana graph with Steiner points constructed as k-means cluster centroids of the dataset vectors. Given a dataset of $n$ vectors in $\mathbb{R}^d$, we run spherical k-means (using FAISS) with $k \in \{0.05n, 0.15n, 0.25n\}$ clusters and 20 Lloyd iterations, producing $k$ centroid vectors that serve as routing-only nodes. For cosine-metric datasets, we use spherical k-means, which re-normalizes centroids to unit norm after each iteration; the resulting centroids lie on the unit sphere alongside the data vectors. The centroid vectors are appended to the original dataset, forming an augmented point set of size $n + k$.

We build a Vamana proximity graph over the augmented point set using ParlayANN's parallel implementation with degree bound $R \in \{32, 64\}$, pruning parameter $\alpha = 1.2$, build beam width $L = 128$, and two construction passes. Crucially, the Steiner points participate in the graph as first-class nodes: they receive incoming and outgoing edges during Vamana's greedy-insert procedure, and they consume edge slots in the degree-$R$ budget just as data points do. The baseline graph is built identically on only the $n$ original points, ensuring that any performance difference is attributable to the Steiner augmentation rather than graph construction artifacts.

At query time, we run beam search with beam width $W$ on the augmented graph. The search maintains a priority queue (the beam) of at most $W$ candidate nodes sorted by similarity to the query $q$. At each step, the closest unexpanded node in the beam is expanded: all its neighbors in the graph are scored against $q$ (via inner product for cosine, or negative squared Euclidean distance), and any node closer than the current $W$-th best candidate is inserted into the beam. Steiner nodes participate fully in the beam—they can be expanded and their edges traversed—but they are excluded from the result set. The search terminates when all nodes in the beam have been expanded.

We evaluate each configuration across a geometric progression of beam widths $W = 4, 8, 16, 32, 64, 96, 144, 216, \ldots$, doubling (and later scaling by $1.5\times$) until the total number of distance computations exceeds a fraction $f = 0.1$ of the dataset size. For each beam width, we record Recall@1, Recall@10, and Recall@100 against brute-force ground truth computed under the same metric, as well as the average number of distance computations per query. We report distance computations to visible (data) nodes separately from those to Steiner nodes, since the latter represent routing overhead that does not directly contribute to result quality. All recall curves plot recall against visible distance computations, enabling a fair comparison of the computational cost required to achieve a given recall level.
